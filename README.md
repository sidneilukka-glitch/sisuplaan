# Sisuplaan — Full Source

Complete source code for the Sisuplaan content planning app. Built with TanStack Start v1, React 19, Tailwind v4, and Supabase.

## Quick start (local)

```bash
bun install      # or: npm install
cp .env.example .env   # fill in values
bun dev          # http://localhost:8080
```

## Required environment variables

```
VITE_SUPABASE_URL=
VITE_SUPABASE_PUBLISHABLE_KEY=
VITE_SUPABASE_PROJECT_ID=
LOVABLE_API_KEY=          # used by /api/calendar, /api/repurpose, /api/strategy
```

If you are deploying without Lovable AI Gateway, replace `LOVABLE_API_KEY` calls in `src/routes/api.*.ts` with a direct provider (OpenAI / Gemini / Anthropic). See `sisuplaan-deploy-bundle.zip` for ready-made patches.

## Deploy to Vercel

1. Push this folder to a new GitHub repo.
2. Import the repo in Vercel.
3. Replace `vite.config.ts` with the Vercel-targeted version from `sisuplaan-deploy-bundle.zip` (changes the TanStack Start target from `cloudflare` to `vercel` and removes the Cloudflare plugin).
4. Add the environment variables above in Vercel project settings.
5. Deploy.

## Deploy to Cloudflare (default)

The project ships configured for Cloudflare Workers via `wrangler.jsonc`.
`bun run build` produces a Worker bundle ready for `wrangler deploy`.

## Notes

- `src/routeTree.gen.ts` is auto-generated by the TanStack Router Vite plugin on first `dev`/`build`. Do not commit it manually.
- `src/integrations/supabase/client.ts` and `types.ts` are kept as-is — regenerate types via Supabase CLI if your schema changes.
- Database schema lives in your Supabase project (project ref in `.env`). Migrations are managed through Supabase.
